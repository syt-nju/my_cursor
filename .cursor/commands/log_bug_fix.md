# Log Bug Fix

When I ask you to log a bug fix or document a resolved issue, follow these steps and save the resulting document to the `syt_docs/bug_fix` directory. Create the directory if it does not exist. Please name the file appropriately based on the bug or issue.

1. **Document the Phenomenon** - Clearly describe the bug and its symptoms:
    - Exact error messages or stack traces
    - Steps to reproduce the issue
    - Expected behavior vs actual behavior
    - Affected files, components, or modules

2. **Analyze the Root Cause** - Explain why the bug occurred:
    - Logical flaws, data inconsistencies, or missed edge cases
    - Any unexpected interactions between systems

3. **Detail the Solution** - Describe how the bug was fixed:
    - Specific code changes made
    - Why this approach was chosen
    - How it prevents the issue from happening again

4. **Review and Confirm** - Summarize the fix and confirm if any further testing or related updates (e.g., updating tests or documentation) are needed.

Make sure the log is detailed enough for future reference, yet concise and easy to read.
